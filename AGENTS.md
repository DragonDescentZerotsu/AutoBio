## MuJoCo / AutoBio notes
- `autobio/README.md` asks for a separate `autobio` conda env with Python 3.11 and `mujoco==3.3.0`.
- The bundled `autobio/libmjlab.so.3.3.0` depends on `libmujoco.so.3.3.0`; it is expected to work in that env.
- Do not binary-patch `libmjlab.so.3.3.0` for MuJoCo 3.8.x. It can load after patching but segfaults during model compilation because the plugin ABI/behavior is not compatible enough.
- For quick pipette visualization in the existing `aero_sim` env, use `AUTOBIO_SKIP_MJLAB=1` with `autobio/model/scene/mani_pipette_viewer.xml`. That viewer-only scene replaces the screw-threaded 50 ml tube with the non-plugin `centrifuge_50ml.gen.xml`; it is not equivalent for screw/unscrew physics.
- `AUTOBIO_SKIP_MJLAB=1` is also supported by `screw_loosen.py` and `screw_tighten.py`, so full screw tasks can be compiled against a manually loaded replacement plugin.
- `mjlab.sdf.thread` implements helical thread SDFs configured by `pitch`, `radius`, `low`, `high`, and `gauge`. `AutoBio.pdf` gives the helix SDF approximation in appendix A.2, but the repo currently includes only the compiled plugin, not its source.
- A fresh `autobio` conda env was created at `/data/tianang/miniconda/envs/autobio` with Python 3.11 and README dependencies. In that env, the original bundled `libmjlab.so.3.3.0` loads and original `mani_pipette` compiles/resets/steps successfully when `LD_LIBRARY_PATH` includes the env's `site-packages/mujoco`.
- Experimental source for a MuJoCo 3.8.1 `mjlab.sdf.thread` reimplementation lives in `autobio/plugins/mjlab_sdf_thread/`. Build output is `autobio/libmjlab_thread.so.<version>` and is ignored by git. This reimplementation lets `mani_pipette` compile/reset/step in `aero_sim`.
- The screw-thread reimplementation now uses the paper's z-axis helix convention, the original bounded-helix branch behavior inferred from the exported `thread_distance_and_gradient` symbol, and the original `1e-8` finite-difference gradient step. In MuJoCo 3.3.0 it matches the original plugin's generated `model/scene/screw_test.xml` SDF mesh vertex/face counts, bounds, mesh positions, and mesh quaternions. Use `autobio/plugins/mjlab_sdf_thread/compare_screw_test.py` for behavior regression. Current default 2000-step 3.3.0 comparison: `ctrl=1` original yaw/dz/force `0.5075/0.000172/1.7502`, reimplementation `0.5075/0.000172/1.7500`; `ctrl=-1` original `-1.2978/-0.000980/7.6780`, reimplementation `-1.2981/-0.000971/7.3911`. Remaining differences are small in the 3.3.0 isolated screw test; MuJoCo 3.8.1 may still differ because the contact solver changed.
